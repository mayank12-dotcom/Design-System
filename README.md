# EDEN Design System

This repo bootstraps the EDEN design system from `eden-ui-tokens.json`. That token file is the only source of truth for token values, alias relationships, and theme modes.

## What is included

- Token normalization from the Figma-exported JSON source
- Generated CSS custom properties and TypeScript token exports
- Light and dark theme contracts
- React + TypeScript button-family components
- Standalone documentation site for tokens and components under `docs/site`
- Storybook stories that mirror the first-wave Figma button matrix
- Vitest coverage for token generation and component state contracts

## Repo structure

- `eden-ui-tokens.json`: source of truth for all tokens
- `scripts/build-tokens.mjs`: token ingestion and generation pipeline
- `scripts/build-docs-token-data.mjs`: docs token data generator
- `src/tokens/generated`: generated raw, CSS, and TypeScript token artifacts
- `src/components`: first-wave button family components
- `src/theme`: theme-mode contract and wrapper
- `stories`: Storybook coverage for the button family
- `docs/site`: static design-system documentation site

## Token workflow

1. Update `eden-ui-tokens.json`.
2. Run `npm run build:tokens`.
3. Consume tokens through generated CSS variables or the exported TypeScript contract.

Do not add manual token values in component styles. Components should consume the generated semantic variables first, and only fall back to primitive variables when the source file does not define a semantic token for that use case.

## Documentation site

The repo includes a static docs app in `docs/site` for browsing:

- token collections from `eden-ui-tokens.json`
- semantic and primitive color groups
- spacing and radius scales
- responsive typography tokens
- first-wave component references

The docs app is interactive and supports filtering, group collapse, deep-linking, and token copy behavior. The token payload is generated by:

1. updating `eden-ui-tokens.json`
2. running `npm run build:docs:data`
3. opening `docs/site/index.html` through a static server

See `docs/README.md` for the docs workflow.

## Figma and GitHub sync

The repo is linked to the EDEN Figma file through `figma.config.json`. The working flow is documented in `docs/FIGMA_GITHUB_WORKFLOW.md`.

- Figma changes flow into GitHub through `eden-ui-tokens.json` plus `npm run sync:figma:pull`
- GitHub changes flow back to Figma through the mapped component nodes in `src/code-connect/mappings.ts`
